# DoctorLink 백엔드 서버

## 서버 실행 방법

### 1. 백엔드 디렉토리로 이동
```bash
cd backend
```

### 2. 서버 시작
```bash
npm run dev
```

서버가 정상적으로 시작되면 다음과 같은 메시지가 표시됩니다:
```
🚀 DoctorLink API 서버가 포트 3000에서 시작되었습니다.
🌍 모든 IP에서 접근 가능: http://0.0.0.0:3000
💚 헬스 체크: http://localhost:3000/api/health
🔑 테스트 계정: test@test.com / test123
```

### 3. 서버 상태 확인
브라우저나 다른 터미널에서 다음 URL로 접속하여 서버가 정상 작동하는지 확인할 수 있습니다:
```
http://localhost:3000/api/health
```

정상적으로 작동하면 다음과 같은 JSON 응답을 받게 됩니다:
```json
{
  "success": true,
  "message": "DoctorLink API 서버가 정상적으로 작동 중입니다.",
  "data": {
    "status": "healthy",
    "timestamp": "2025-01-...",
    "version": "1.0.0"
  }
}
```

## 주의사항

- 백엔드 서버는 **인메모리 저장소**를 사용합니다. 서버를 재시작하면 모든 데이터가 초기화됩니다.
- 프론트엔드 앱을 실행하기 **전에** 백엔드 서버를 먼저 시작해야 합니다.
- 포트 3000이 이미 사용 중인 경우, 다른 프로그램을 종료하거나 `.env` 파일에서 포트를 변경할 수 있습니다.

## 문제 해결

### 서버가 시작되지 않을 경우
1. Node.js가 설치되어 있는지 확인: `node --version`
2. 의존성이 설치되어 있는지 확인: `npm install`
3. 포트 3000이 사용 중인지 확인하고 필요시 다른 포트 사용

### 프론트엔드에서 연결이 안 될 경우
1. 백엔드 서버가 실행 중인지 확인
2. 방화벽 설정 확인
3. `src/screens/admin/` 폴더의 파일들에서 API URL이 `http://localhost:3000`로 설정되어 있는지 확인
